# How I structure Node.js microservices in production

After a few years building back-ends that must scale, some patterns keep showing up. Here are the ones that help me the most.

## Clear boundaries

Each service owns its data. Communication via explicit contracts — never share a table between services.

## Observability from day zero

Structured logs, metrics and distributed tracing. Without them, debugging in production becomes guesswork.

> Code is only half the job — operating it well is the other half.
